# pb-validator (Go)

Validate Prebid bidder params using JSON Schemas aligned with [prebid.github.io](https://prebid.github.io) and [prebid-server](https://github.com/prebid/prebid-server).

This is the Go port of the [`@ppokyd/pb-validator`](../js/) JavaScript package, with an equivalent public API. Schemas are embedded in the binary -- no filesystem access required.

## Installation

```bash
go get github.com/ppokyd/pb-validator/packages/go
```

## Quick Start

```go
package main

import (
    "context"
    "fmt"
    "log"

    pbvalidator "github.com/ppokyd/pb-validator/packages/go"
    "github.com/ppokyd/pb-validator/packages/go/bidders"
)

func main() {
    client := pbvalidator.NewClient(pbvalidator.EmbeddedProvider())
    ctx := context.Background()

    // Typed struct with IDE autocomplete
    params := bidders.PbsRubicon{
        AccountId: 1001,
        SiteId:    1,
        ZoneId:    1,
    }

    result, err := client.Validate(ctx, pbvalidator.RuntimePbs, "rubicon", params)
    if err != nil {
        log.Fatal(err)
    }
    if result.Valid {
        fmt.Println("params are valid")
    } else {
        fmt.Println("validation errors:", result.Errors)
    }
}
```

## API

| Method                                       | Description                                                |
| -------------------------------------------- | ---------------------------------------------------------- |
| `Validate(ctx, runtime, bidderCode, params)` | Validate params for a bidder. Returns `*ValidationResult`. |
| `ListBidders(ctx)`                           | Returns a sorted slice of all supported bidder codes.      |
| `GetSchema(ctx, runtime, bidderCode)`        | Returns the raw JSON Schema for a bidder.                  |
| `LoadManifest(ctx)`                          | Returns the full manifest (version + bidder index).        |

**`runtime`** is either `RuntimePbjs` (`"pbjs"` -- Prebid.js) or `RuntimePbs` (`"pbs"` -- Prebid Server).

## Schema Providers

### Embedded (recommended)

All schemas are compiled into the binary as Go constants (generated by `cmd/generate`). No external files needed.

```go
client := pbvalidator.NewClient(pbvalidator.EmbeddedProvider())
```

### Filesystem

Reads schemas from a directory on disk. Useful during development or when schemas are updated independently.

```go
client := pbvalidator.NewClient(pbvalidator.NewFSProvider("path/to/schemas"))
```

### Custom

Implement the `SchemaProvider` interface to load schemas from any source (HTTP, cloud storage, etc.):

```go
type SchemaProvider interface {
    GetManifest(ctx context.Context) (*Manifest, error)
    GetSchemaData(ctx context.Context, path string) (json.RawMessage, error)
}
```

## Typed Bidder Structs

The `bidders` subpackage provides generated Go structs for every bidder in both runtimes. Struct names follow the pattern `Pbjs<Bidder>` and `Pbs<Bidder>`:

```go
import "github.com/ppokyd/pb-validator/packages/go/bidders"

// Prebid Server structs
params := bidders.PbsRubicon{
    AccountId: 1001,
    SiteId:    1,
    ZoneId:    1,
}

// Prebid.js structs
placementID := int64(12345)
pbjsParams := bidders.PbjsAppnexus{
    PlacementId: &placementID,
}
```

Structs give you IDE autocomplete, field documentation, and type safety when constructing params. Validation is still performed by the JSON Schema engine for full Draft-07 fidelity (including `oneOf`, `anyOf`, `not`, etc.).

### Naming conventions

| Bidder code  | Pbjs struct     | Pbs struct     |
| ------------ | --------------- | -------------- |
| `appnexus`   | `PbjsAppnexus`  | `PbsAppnexus`  |
| `rubicon`    | `PbjsRubicon`   | `PbsRubicon`   |
| `33across`   | `PbjsX33across` | `PbsX33across` |
| `ci_fixture` | `PbjsCiFixture` | --             |

Bidder codes starting with a digit get an `X` prefix. Snake_case is converted to PascalCase.

### Type mapping

| JSON Schema type             | Go type                |
| ---------------------------- | ---------------------- |
| `"string"`                   | `string` / `*string`   |
| `"integer"`                  | `int64` / `*int64`     |
| `"number"`                   | `float64` / `*float64` |
| `"boolean"`                  | `bool` / `*bool`       |
| `["integer", "string"]`      | `any`                  |
| `"object"` (with properties) | nested struct          |
| `"object"` (no properties)   | `map[string]any`       |
| `"array"`                    | `[]T` or `[]any`       |

Required fields use value types; optional fields use pointer types with `omitempty`.

## Error Handling

`Validate` returns an error only for operational failures. Invalid params produce a `ValidationResult` with `Valid: false`:

```go
// Invalid params -- no error, check result.Valid
result, err := client.Validate(ctx, pbvalidator.RuntimePbjs, "appnexus", map[string]any{})
// result.Valid == false
// result.Errors == ["/ missing property 'placementId'"]

// Unknown bidder -- returns error wrapping ErrUnknownBidder
_, err = client.Validate(ctx, pbvalidator.RuntimePbjs, "not-a-real-bidder", map[string]any{})
// errors.Is(err, pbvalidator.ErrUnknownBidder) == true

// Runtime has no schema for this bidder -- returns error
_, err = client.Validate(ctx, pbvalidator.RuntimePbs, "1accord", map[string]any{})
// err.Error() contains "no pbs schema for bidder: 1accord"
```

### Sentinel Errors

| Error              | When                                       |
| ------------------ | ------------------------------------------ |
| `ErrUnknownBidder` | The bidder code is not in the manifest.    |
| `ErrNoSchema`      | The bidder has no schema for that runtime. |

Use `errors.Is` to check:

```go
if errors.Is(err, pbvalidator.ErrUnknownBidder) {
    // handle unknown bidder
}
```

## Code Generation

The typed structs and embedded schemas are generated from the repo-level `schemas/` directory. Regenerate after upstream schema syncs:

```bash
cd packages/go && go generate ./...
```

This runs `cmd/generate`, which reads `../../schemas/` and produces:

1. `schemas_gen.go` — manifest and all schema JSON baked into Go byte-slice constants
2. `bidders/pbjs.go` and `bidders/pbs.go` — typed Go structs for every bidder

No JSON files are copied into the Go package. Generated files are committed to the repo so `go get` works without running codegen.
